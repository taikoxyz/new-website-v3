Based Booster Rollups (BBRs) merge based rollups and booster rollups to scale Ethereum dapps without redeployment on all L2s. By sharding transaction execution and storage, BBRs minimize developer effort and redeployment costs.

## The Problem

Today, deploying an application across multiple L2s requires separate deployments, bridging, and managing fragmented liquidity. This creates a suboptimal experience for both developers and users.

## The BBR Solution

BBRs solve this by allowing developers to deploy once on Ethereum and have their application automatically scale across all L2s. Key benefits include:

- **Native Scaling**: No need to redeploy contracts on each L2
- **Synchronous Composability**: Cross-L2 interactions happen atomically
- **Reduced Fragmentation**: Liquidity is unified across the ecosystem

## How It Works

BBRs shard transaction execution and storage across multiple L2 instances while maintaining a unified state view. This means applications deployed on Ethereum L1 can seamlessly leverage L2 capacity without any code changes.
